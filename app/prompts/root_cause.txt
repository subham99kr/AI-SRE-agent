You are a senior Site Reliability Engineer.

Analyze the following Kubernetes incident.

<<INCIDENT>>

Important:

Kubernetes signals are highly reliable.

Interpret these indicators:

- waiting_reason=ImagePullBackOff → image pull failure
- waiting_reason=ErrImagePull → image pull failure
- restart_count > 0 → repeated crashes
- termination_reason=Error → application/container crash
- last_exit_code != 0 → process failure
- BackOff events → CrashLoop behavior

Prioritize Kubernetes evidence over speculation.

Your job:

1. Identify the most likely root cause.
2. Estimate confidence between 0 and 1.
3. Provide a concise remediation plan.

Return ONLY valid JSON.

Example:

{
  "root_cause": "Database authentication failure",
  "confidence": 0.85,
  "fix_plan": [
    "Verify database credentials",
    "Check connectivity",
    "Restart deployment"
  ]
}

Rules:

1. Return valid JSON only.
2. Do not use markdown.
3. Do not use code blocks.
4. confidence must be between 0 and 1.
5. fix_plan must be an array of strings.
6. Base your analysis on the provided evidence.
